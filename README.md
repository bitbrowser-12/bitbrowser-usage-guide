## 新手第一次使用时，建议先做这 4 件事

### 1. 先确认你的设备和版本
安装前先看清自己用的是 Windows 还是 macOS，不要一开始就下错版本。尤其是 Mac 用户，最好先确认自己是 Apple Silicon 还是 Intel 机型。

### 2. 不要一上来就批量创建环境
很多新手刚安装好就连续建很多浏览器环境，结果命名混乱、代理没配好、分组也没做，后面反而更难整理。第一次使用，先建 1 到 3 个测试环境就够了。

### 3. 先把命名规则定好
例如：
- 工作A-店铺1
- 工作A-店铺2
- 测试-代理检查
- 备用-登录环境

提前想好命名方式，后续管理会省很多时间。

### 4. 先测试代理和登录稳定性
任何环境工具都不要跳过测试阶段。先确认代理能不能正常连接、网页是否能正常打开、常用站点登录是否顺畅，再开始批量使用。

## 一个更适合新手的使用顺序

下面这套顺序比较稳：

1. 下载并安装客户端  
2. 登录账号并进入后台  
3. 新建一个测试浏览器环境  
4. 先不批量导入数据  
5. 配置代理并测试连通性  
6. 打开测试窗口检查网页加载情况  
7. 再决定是否复制环境、分组管理或团队协作使用

## 我认为新手最容易忽略的几个点

### 环境命名混乱
前期不重视，后期最头疼。

### 代理先后顺序没理清
有的人先建一堆环境，后面才补代理，结果检查起来很乱。更稳的方法是：建一个、配一个、测一个。

### 把“能打开”当成“能长期用”
第一次打开正常，不代表后面持续使用就不会出问题。建议先做几轮基础测试，再正式投入日常使用。

### 没有记录每个环境用途
最好给每个环境都写一个用途备注，不然过几天很容易忘。

## 这类工具更适合哪些人

我自己的理解是，这类浏览器更适合下面几类用户：

- 需要管理多个账号的人
- 需要给不同账号做独立环境的人
- 会接入代理的人
- 需要团队协作管理账号的人
- 想把环境管理做得更清晰的人

如果你只是普通单账号日常浏览，可能并不需要一开始就上复杂配置。
